## 0 · 一句话结论

**架构底子稳,无"正在暴雷"的洞**(已实测线上 mock-auth 越权返 401,确认 prod 配置正确)。但内测前有 **5 个必修项**,核心矛盾两条:
1. **`.md` 跨用户隔离靠 prompt 自律,不是确定性隔离** —— 产品第一铁律的防线有缝
2. **6-8 分钟长链路是进程内 fire-and-forget,不可恢复** —— 最可能的故障是"用户提交成功但房间一直没简报"

**双模型价值印证**:Codex 抓到 3 个 Claude 漏判/判轻的真问题(`.md` 经 domains/label 泄露、`user_hard_blocklist` 空壳 = 铁律1 未实现、pipeline 重跑实为 no-op),并纠正了 Claude"desensitize 代码级安全""pipeline 幂等可重跑"两个过度自信的结论。

## 2 · P0 · 内测前必修

### P0-1 · `.md` 跨用户隔离靠 prompt 自律,非确定性隔离 🔴
**铁律3 防线有缝。两条子路径:**

**(a) 客户端可控字段 → desensitize → hook_text 给对方看**
`md/schemas.py:19` 定义了 `ALLOWED_DOMAINS` 但**从未使用**;`ProfileDomains.interested/avoided` 是裸 `list[str]`、`TaggedDimension.label` 是裸 `str`,**后端零校验**。`_extract_safe_profile_summary`(`desensitize.py:84`)把这些原样喂进脱敏 LLM → hook_text 进对方可见的 room hint。
**攻击路径**:用户把微信号 / `.md` 原文 / prompt 注入塞进 `domains.interested`,匹配后诱导 hook 暴露给对方。
> ⚠️ Claude 初判"desensitize 输入只有字段路径、代码级安全"是**错的** —— 只查了 `source_segments`(确为路径),漏了 domains/labels 是客户端自由文本。Codex 精准命中。

**(b) 宿主 portrait 原文 → agent_chat prompt → utterance 被对方回看**
`agent_chat/engine.py:153` 把宿主完整 `portrait`(含 `body/title_reason/core_tension/debug`)放进 speaker prompt,生成的 utterance 对方能通过 summary / agent_chat 回放看到。只靠 system prompt"不暴露原文"自律,**输出侧零拦截**。

**修法**:① 后端白名单/重算 domains、code、label(非法值丢弃或归一);② 剥离 `portrait.debug`,agent prompt 只给数值桶 + 枚举标签 + 后端生成的短摘要,不给完整 portrait 原文;③ **确定性兜底**:hook_text / utterance 入库前跑 scanner,比对双方 profile 长字符串 / n-gram,命中则丢弃或重跑。

### P0-2 · `user_hard_blocklist` 是空壳 → 铁律1(平台底线封禁)未实现 🔴
`room/models.py:44` 有 `user_hard_blocklist` 表,但**全代码库无任何写入**(已脚本核对)。真人举报只写 `chat_reports` + 可撤销软拉黑;Agent 检测到 `boundary_hit='铁律'` 只结束当场互聊。**黄赌毒 / 违法用户没有独立、不可撤销、不可对用户暴露的封禁闭环** —— 这正是产品第一恐惧("变黄赌毒平台")。
> Claude 完全漏了这条。
**修法**:新增治理写入路径(report / Agent boundary_hit='铁律' / admin 审核 → 写 `user_hard_blocklist` + evidence);match / chat / auth 侧统一过滤 hard-blocked 用户;用户端只显示"不可用",不暴露原因。
**内测豁免判断**:闭门内测(~100 熟人)可先用**人工 ops 兜底**(发现→手动 SQL 封),但**公开发布前这是硬门槛**。

### P0-3 · 配置 fail-open,无启动守卫 🔴(当前未暴雷,零安全网)
`settings.py:41` 所有配置有默认值:`env` 默认 `dev`、`jwt_secret` 默认已知串。`is_dev=True` 时 `deps.py:55` 的 `X-Mock-User-Id` 头可越权成任意用户、cookie 非 Secure、JWT 可伪造。前端 `NEXT_PUBLIC_DEV_MOCK_AUTH` 在 Docker build arg 默认 `true`。
> **实测**:线上 `curl -H "X-Mock-User-Id: 29" /api/auth/me` → `401`,确认 Railway 已设 `ENV=prod`,**当前安全**。但一次 env 丢失(新 deploy / fork / 误清)就**静默全站敞开**,无任何启动报错。两模型都判最高优。
**修法**:`main.py` lifespan 加 fail-fast 断言 —— 非 dev 启动必须 `jwt_secret≠默认` + `admin_secret 非空` + `cors 非 localhost`,否则 crash;mock-auth 改用独立 flag(`ENABLE_MOCK_AUTH`)不搭 `is_dev` 便车;前端 build arg 默认 `false`。

### P0-4 · pipeline 不可恢复,"重跑接着干"实为 no-op 🔴
`match/pipeline.py:43` 全链路是进程内 fire-and-forget BackgroundTask。部署 / 重启 / worker crash / LLM 失败发生在"match 已建、hook/chat/summary 未完成"之后:重跑因 `_existing_match_partners` 把已 match 对象**过滤掉 → `new_matches=[] → 直接 return`**。注释承诺的"幂等可重跑"**不成立**。
> Claude 判"pipeline 幂等"是**错的** —— 那个 dedup 恰恰让重跑变 no-op。Codex 证伪。
**最可能的真实故障**:100 人陆续进入,某些用户后台半路丢 → 房间永远没简报,且无自愈、无恢复入口。
**修法**:① 内测期**禁止 deploy**(最省,先定规矩);② 短期加 admin endpoint 扫"有 match 无 summary"的用户补跑(缺 hook 补 desensitize / 缺 done chat 补 agent_chat / 缺 summary 补 summary,**按 stage 补而非按新 match**);③ 中期落 DB job 表 `pipeline_jobs(user_id, kind, status, stage, attempts, locked_by, last_error)`,`FOR UPDATE SKIP LOCKED` 拉取。

### P0-5 · 后台并发无上限 + LLM 持有连接 → 连接池非真安全 🔴
`4 worker × (10+10) = 80 < 97` **不是安全证明**。后台 pipeline 无并发上限;`desensitize`(`desensitize.py:191`)/ `summary`(`summary/engine.py:281`)做完 SELECT 后进 LLM 调用,**首个长 LLM await 期间仍占着 session**;叠加用户请求 + SSE 收尾写库 + cron + admin,突发时 `pool_timeout=30s` 后大面积 500 / 卡顿。
> 两模型收敛。Codex 战略性修正:**真正的杠杆是限制后台并发,不是继续加 worker**。
**修法**:① LLM gateway 加显式 `timeout=60` + `max_retries=2`(`AsyncOpenAI(...)` 当前默认 600s,5 分钟级 worker stall 的根源);② 后台 pipeline 加全局 semaphore(同时 2-4 个);③ LLM call 期间释放 DB session(参考 `agent_self/engine.py:405` 已正确的 SSE 多 session 模式);④ 可选:pool 降到 `5+5` 或接 PgBouncer。

## 5 · 已检查 · 无问题(两模型一致认可的强项)

- **`.md` 隔离的另几条链是真安全的**:agent_self RAG 三条向量检索全按 host `user_id` 过滤(`rag.py:96/126/191`)不跨用户;summary host 只见自己 .md + 对方 public utterance;SSE 主路径在 LLM stream 前释放 DB session(`engine.py:405`)
- 密码 bcrypt(sha256+base64 预处理,rounds=10,未回退 passlib);login 失败统一 401 + 不存在 user 也跑 dummy bcrypt(防枚举 + 防 timing)
- OAuth state CSRF 校验在;redirect path 固定无开放重定向
- 17 个 admin endpoint 全部有 `_require_admin`(脚本核对,无裸 endpoint)
- 无 SQL 注入(`text()` 仅用于固定 SQL / JSONB 操作,无拼接用户输入)
- 前端无 XSS sink(无 `dangerouslySetInnerHTML`/`eval`),头像 data URL 前后端都拒 SVG,无硬编码 secret,session cookie HttpOnly
- `get_session` `async with` + finally close 无泄漏;matches 有 pair unique index;polling 有 cleanup
- pipeline 每步独立 session + 失败 print-continue 隔离;BackgroundTask 绑定请求 worker 不重复跑
